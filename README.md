# convex-nextjs-chatbot-kit

A production-ready, open-source starter kit for building AI chatbot applications using **Next.js 16 App Router**, **Convex** (database + backend), **Convex Auth** (email OTP), and **Convex Persistent Text Streaming**.

This is not a demo or toy. It is a real foundation — with secure auth-gated APIs, real-time sync across tabs, durable streaming that survives page refreshes, and a clean, minimal UI. Fork it and ship.

---

## Why This Stack

| Problem | Solution |
|---|---|
| Streaming responses lost on refresh | **Convex Persistent Text Streaming** stores tokens to Convex as they arrive — resume anywhere |
| Auth-less APIs leaking user data | All queries/mutations gated by `getAuthUserId()` with IDOR ownership checks |
| Complex backend setup | Convex provides serverless functions + realtime database + file storage in one deployment |
| Boilerplate overload | Everything wired: auth, threads, messages, streaming pipeline, UI |

### Streaming Pipeline

```
User sends message
       │
       ▼
sendMessage mutation
  ├── Inserts user message
  ├── Creates empty assistant message
  ├── streamingComponent.createStream() → streamId stored on message
  └── Schedules title generation

       │ (client receives assistantId + responseStreamId via Convex reactivity)
       ▼

ServerMessage component (SSR disabled)
  └── useStream(streamId, POST /chat-stream)
        │
        ▼
chatStream HTTP action
  ├── Auth check (ctx.auth.getUserIdentity)
  ├── Loads thread context from Convex
  ├── Streams from OpenAI
  └── streamingComponent.stream() → appends tokens to Convex in real time

       │ tokens flow back through useStream
       ▼
Text renders token-by-token in the UI
On stream completion → patchAssistantTextClient saves final text to message
```

## Features

- **Persistent streaming** — tokens durably stored in Convex as they stream; page refresh resumes the response
- **Real-time thread sync** — open two tabs; messages appear instantly in both
- **Email OTP auth** — no passwords; 8-digit codes via Resend
- **Secure by default** — every Convex function verifies identity and thread ownership (IDOR-safe)
- **Auto thread titles** — generated by the LLM after the first message
- **Configurable system prompt** — set `SYSTEM_PROMPT` env var, no code changes needed
- **Pluggable model** — set `OPENAI_MODEL` to any OpenAI-compatible model ID
- **Dark/light theme** — Vercel/Linear-inspired design system
- **Streaming cursor animation** — smooth cursor while tokens arrive
- **Thread deletion** — cascading delete of all thread messages

## Quick Start

### 1. Clone

```bash
git clone https://github.com/the-niresh/convex-nextjs-chatbot-kit.git
cd convex-nextjs-chatbot-kit
npm install
```

### 2. Initialize Convex

```bash
npx convex dev
```

This will prompt you to log in, create a Convex project, and write `NEXT_PUBLIC_CONVEX_URL` to `.env.local` automatically.

### 3. Configure Convex Auth

In your [Convex Dashboard](https://dashboard.convex.dev/) → your project → **Settings → Environment Variables**:

| Variable | Value |
|---|---|
| `SITE_URL` | `http://localhost:3000` (dev) or your Vercel URL (prod) |
| `RESEND_API_KEY` | Your Resend API key |

### 4. Configure OpenAI + App

In the same Convex dashboard environment variables:

| Variable | Value |
|---|---|
| `OPENAI_API_KEY` | Your OpenAI API key |
| `OPENAI_MODEL` | `gpt-4o-mini` (default) |
| `SYSTEM_PROMPT` | Your custom system prompt (optional) |
| `NEXT_PUBLIC_APP_URL` | Your public app URL (for CORS + SEO) |

### 5. Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) — sign in with your email, start chatting.

## Customization

### Change the AI persona

Set `SYSTEM_PROMPT` in Convex environment variables:

```
SYSTEM_PROMPT=You are a senior DevOps engineer. Answer concisely and precisely with code examples.
```

### Change the model

```
OPENAI_MODEL=gpt-4o
```

Any OpenAI-compatible model ID works. For non-OpenAI providers, update the client initialization in `convex/streaming.ts`.

### Add OAuth (GitHub, Google)

```ts
// convex/auth.ts
import { GitHub } from "@convex-dev/auth/providers/GitHub";
import { Google } from "@convex-dev/auth/providers/Google";

export const { auth, signIn, signOut, store } = convexAuth({
  providers: [ResendOTP, GitHub, Google],
});
```

See the [Convex Auth docs](https://labs.convex.dev/auth/config/oauth) for full setup.

## Security Model

| Layer | Mechanism |
|---|---|
| Authentication | `getAuthUserId(ctx)` called at the start of every protected function |
| Authorization | Thread ownership checked: `thread.userId === authenticatedUserId` prevents IDOR |
| HTTP action auth | `/chat-stream` validates `ctx.auth.getUserIdentity()` before processing |
| Internal mutations | `patchAssistantText` is `internalMutation` — not callable from clients |
| CORS | Streaming endpoint scoped to `NEXT_PUBLIC_APP_URL` in production |
