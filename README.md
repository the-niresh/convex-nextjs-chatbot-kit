# convex-nextjs-chatbot-kit

A production-ready, open-source starter kit for building AI chatbot applications using **Next.js 16 App Router**, **Convex** (database + backend), **Convex Auth** (email OTP), and **Convex Persistent Text Streaming**.

This is not a demo or toy. It is a real foundation вҖ” with secure auth-gated APIs, real-time sync across tabs, durable streaming that survives page refreshes, and a clean, minimal UI. Fork it and ship.

---

## Why This Stack

| Problem | Solution |
|---|---|
| Streaming responses lost on refresh | **Convex Persistent Text Streaming** stores tokens to Convex as they arrive вҖ” resume anywhere |
| Auth-less APIs leaking user data | All queries/mutations gated by `getAuthUserId()` with IDOR ownership checks |
| Complex backend setup | Convex provides serverless functions + realtime database + file storage in one deployment |
| Boilerplate overload | Everything wired: auth, threads, messages, streaming pipeline, UI |

### Streaming Pipeline

```
User sends message
       в”Ӯ
       в–ј
sendMessage mutation
  в”ңв”Җв”Җ Inserts user message
  в”ңв”Җв”Җ Creates empty assistant message
  в”ңв”Җв”Җ streamingComponent.createStream() вҶ’ streamId stored on message
  в””в”Җв”Җ Schedules title generation

       в”Ӯ (client receives assistantId + responseStreamId via Convex reactivity)
       в–ј

ServerMessage component (SSR disabled)
  в””в”Җв”Җ useStream(streamId, POST /chat-stream)
        в”Ӯ
        в–ј
chatStream HTTP action
  в”ңв”Җв”Җ Auth check (ctx.auth.getUserIdentity)
  в”ңв”Җв”Җ Loads thread context from Convex
  в”ңв”Җв”Җ Streams from OpenAI
  в””в”Җв”Җ streamingComponent.stream() вҶ’ appends tokens to Convex in real time

       в”Ӯ tokens flow back through useStream
       в–ј
Text renders token-by-token in the UI
On stream completion вҶ’ patchAssistantTextClient saves final text to message
```

## Features

- **Persistent streaming** вҖ” tokens durably stored in Convex as they stream; page refresh resumes the response
- **Real-time thread sync** вҖ” open two tabs; messages appear instantly in both
- **Email OTP auth** вҖ” no passwords; 8-digit codes via Resend
- **Secure by default** вҖ” every Convex function verifies identity and thread ownership (IDOR-safe)
- **Auto thread titles** вҖ” generated by the LLM after the first message
- **Configurable system prompt** вҖ” set `SYSTEM_PROMPT` env var, no code changes needed
- **Pluggable model** вҖ” set `OPENAI_MODEL` to any OpenAI-compatible model ID
- **Dark/light theme** вҖ” Vercel/Linear-inspired design system
- **Streaming cursor animation** вҖ” smooth cursor while tokens arrive
- **Thread deletion** вҖ” cascading delete of all thread messages

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/the-niresh/convex-nextjs-chatbot-kit.git
cd convex-nextjs-chatbot-kit
npm install
```

### 2. Start Convex (creates your project + writes `.env.local`)

```bash
npx convex dev
```

Leave this running вҖ” it watches your `convex/` folder and hot-reloads functions.

### 3. Generate Auth keys

In a new terminal:

```bash
node generateKeys.mjs
```

This outputs `JWT_PRIVATE_KEY` and `JWKS`. Copy both values вҖ” you'll paste them into the Convex dashboard next.

### 4. Set environment variables in Convex Dashboard

Go to [dashboard.convex.dev](https://dashboard.convex.dev) вҶ’ your project вҶ’ **Settings вҶ’ Environment Variables** and add:

| Variable | Value |
|---|---|
| `JWT_PRIVATE_KEY` | Output from `generateKeys.mjs` |
| `JWKS` | Output from `generateKeys.mjs` |
| `SITE_URL` | `http://localhost:3000` (dev) or your Vercel URL (prod) |
| `AUTH_RESEND_KEY` | Your [Resend](https://resend.com) API key |
| `OPENAI_API_KEY` | Your OpenAI API key (or OpenRouter key вҖ” see below) |
| `OPENAI_MODEL` | `gpt-4o-mini` (default, optional) |
| `SYSTEM_PROMPT` | Custom system prompt (optional) |

### 5. Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) вҖ” sign in with your email, start chatting.

## Customization

### Change the AI persona

Set `SYSTEM_PROMPT` in Convex environment variables:

```
SYSTEM_PROMPT=You are a senior DevOps engineer. Answer concisely and precisely with code examples.
```

### Change the model

```
OPENAI_MODEL=gpt-4o
```

### Use OpenRouter (or any OpenAI-compatible provider)

[OpenRouter](https://openrouter.ai) gives you access to hundreds of models (Claude, Gemini, Mistral, Llama, etc.) through one API key. Set these two variables in the Convex dashboard:

| Variable | Value |
|---|---|
| `OPENAI_API_KEY` | Your OpenRouter API key |
| `OPENAI_BASE_URL` | `https://openrouter.ai/api/v1` |
| `OPENAI_MODEL` | Any model slug, e.g. `anthropic/claude-3-5-sonnet` |

No code changes needed вҖ” the app reads `OPENAI_BASE_URL` automatically.

### Add OAuth (GitHub, Google)

```ts
// convex/auth.ts
import { GitHub } from "@convex-dev/auth/providers/GitHub";
import { Google } from "@convex-dev/auth/providers/Google";

export const { auth, signIn, signOut, store } = convexAuth({
  providers: [ResendOTP, GitHub, Google],
});
```

See the [Convex Auth docs](https://labs.convex.dev/auth/config/oauth) for full setup.

## Deployment

### Vercel

1. Push to GitHub
2. Import to [Vercel](https://vercel.com/new)
3. Add environment variables: `NEXT_PUBLIC_CONVEX_URL`, `NEXT_PUBLIC_CONVEX_SITE_URL`, `NEXT_PUBLIC_APP_URL`
4. In Convex dashboard, update `SITE_URL` to your Vercel deployment URL
5. Deploy

### Convex Production

```bash
npx convex deploy
```

Deploys your Convex functions to the production environment.

## Security Model

| Layer | Mechanism |
|---|---|
| Authentication | `getAuthUserId(ctx)` called at the start of every protected function |
| Authorization | Thread ownership checked: `thread.userId === authenticatedUserId` prevents IDOR |
| HTTP action auth | `/chat-stream` validates `ctx.auth.getUserIdentity()` before processing |
| Internal mutations | `patchAssistantText` is `internalMutation` вҖ” not callable from clients |
| CORS | Streaming endpoint scoped to `NEXT_PUBLIC_APP_URL` in production |
